## 🔑 核心概念详解

### 1. 三层导演逻辑

#### 第一层：全局逻辑层（一致性保证）

**目的**：防止"角色变脸"和"环境跳戏"

**包含内容**：
```python
global_settings = {
    "main_character": "核心角色设定",
    "environment": "环境设定",
    "lighting_tone": "光影基调",
    "visual_style": "视觉风格"
}
```

**示例**：
- 核心角色：长发黑衣少女，眼神清冷，气质高冷
- 环境：赛博朋克城市，霓虹灯闪烁，雨夜
- 光影：冷蓝色调，高对比度，戏剧化光效

#### 第二层：导演编剧层（标准化剧本）

**目的**：标准化输出，便于生成

**6 要素**：
```python
segment = {
    "shot_id": "Shot 01",          # 镜号
    "setting": "场景描述",         # 场景
    "camera": "运镜描述",          # 运镜
    "visual_prompt": "完整提示词", # 动态画面描述提示词
    "vo": "旁白内容",              # 旁白
    "sfx": "音效描述"              # 音效
}
```

**示例**：
```python
{
    "shot_id": "Shot 01",
    "setting": "破旧小酒馆 - 深夜",
    "camera": "低角度推近特写",
    "visual_prompt": "电影级画质，长发黑衣少女，眼神清冷...",
    "vo": "她终于回到了这个熟悉又陌生的地方。",
    "sfx": "沉重的木门推开声，风铃轻响"
}
```

#### 第三层：Seedance 2.0 适配层

**目的**：优化提示词，适应 Seedance 2.0

**核心要求**：
- 细腻表情：瞳孔微动、汗水滑落、肌肉颤抖
- 高张力动作：慢动作爆发、碎屑飞溅、剧烈喘息
- 物理交互：水花溅射、烟雾扩散、粒子特效

### 2. call_openai_api_director_v3()

**作用**：调用 OpenAI/DeepSeek API 生成分镜

**位置**：app.py 第 1172 行

**核心逻辑**：
```python
def call_openai_api_director_v3(story_text, api_mode, tension_level, ...):
    # 1. 构建 API 客户端
    client = OpenAI(
        api_key="...",
        base_url="https://api.deepseek.com"
    )

    # 2. 构建系统 Prompt
    system_prompt = DIRECTOR_SYSTEM_PROMPT  # 三层导演逻辑

    # 3. 调用 API
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt}
        ]
    )

    # 4. 解析结果
    result = parse_api_response(response)

    return result
```

**关键点**：
- 系统 Prompt 包含三层导演逻辑
- 自动提取全局设定
- 生成 6 要素标准化输出

### 3. generate_dramatic_storyboard()

**作用**：本地模拟生成分镜（无需 API）

**位置**：app.py 第 1348 行

**核心逻辑**：
```python
def generate_dramatic_storyboard(story_text, tension_level, ...):
    # 1. 分析故事情绪
    emotion_segments = analyze_story_emotion(story_text)

    # 2. 生成每个分镜
    segments = []
    for i, segment in enumerate(emotion_segments):
        # 构建戏剧化描述
        description = build_dramatic_description(
            segment,
            tension_level,
            emotion_filter
        )

        # 生成 6 要素
        shot = {
            "shot_id": f"Shot {i+1}",
            "setting": extract_setting(segment),
            "camera": select_camera_movement(i),
            "visual_prompt": description,
            "vo": generate_vo(segment),
            "sfx": generate_sfx(segment)
        }

        segments.append(shot)

    # 3. 构建最终提示词
    final_prompt = build_final_director_prompt(segments, ...)

    return {
        "segments": segments,
        "final_prompt": final_prompt
    }
```

**优势**：
- 无需 API Key
- 响应速度快
- 适合快速测试

def display_v3_director_result(result, api_mode, story_text, ...):
    # 1. 显示全局设定
    st.metric("核心角色", result["global_settings"]["main_character"])
    st.metric("环境设定", result["global_settings"]["environment"])
    st.metric("光影基调", result["global_settings"]["lighting_tone"])

    # 2. 显示分镜卡片
    for segment in result["segments"]:
        with st.expander(f"🎬 分镜 {segment['shot_id']}"):
            # 显示 6 要素
            st.markdown(f"**⏱️ 时间/场景：** {segment['setting']}")
            st.markdown(f"**🎥 运镜：** {segment['camera']}")
            st.code(segment['visual_prompt'])

            # 显示旁白和音效
            st.success(f"**🎤 旁白：** \"{segment['vo']}\"")
            st.success(f"**🔊 音效：** \"{segment['sfx']}\"")

    # 3. 显示完整提示词
    st.code(result["complete_prompt"])

    # 4. 下载按钮
    st.download_button(...)

## 💡 核心概念总结

### 1. 一致性保证

**问题**：角色变脸、环境跳戏

**解决方案**：
- 全局逻辑层提取并锁定核心设定
- 所有分镜使用相同的外貌特征描述
- 环境和光影保持一致

### 2. 标准化输出

**目的**：便于生成和理解

**6 要素**：
1. Shot ID（镜号）
2. Setting（场景）
3. Camera（运镜）
4. Visual Prompt（动态画面描述提示词）
5. VO（旁白）
6. SFX（音效）

### 3. 张力调节

**3 级：温和** - 平静、温和、低强度
**5 级：中等** - 明显起伏、中等强度
**7 级：强烈** - 紧张刺激、高强度
**10 级：史诗** - 史诗级冲突、最高强度

### 4. AI vs 本地

| 特性 | AI 模式 | 本地模式 |
|------|---------|---------|
| 质量 | 高 | 中等 |
| 速度 | 较慢（2-5s） | 快（<1s） |
| 成本 | 需要付费 | 免费 |
| API Key | 需要 | 不需要 |
